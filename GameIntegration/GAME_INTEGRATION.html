Overview

Step-by-step recipe for wiring EOSpro into a real multiplayer game project on UE 4.27. Covers all five Blueprint classes you'll touch — UGameInstance, AGameMode, APlayerController, the HUD UMG, and a USaveGame subclass — plus the cross-platform purchase bridge and the Anti-Cheat hookup.

Companion docs This is the integration cookbook. The 262-node BP reference is in ../index.html.

TL;DR — minimum-viable flow

GameInstance::Init() ─► EOS_Platform_Create (UEOSproSubsystem auto-init) │ PlayerController::BeginPlay ─► EOSLoginWithPlatform(EpicGames) │ (or DevAuth in PIE, Steam/Apple/Google in shipping) ▼ [Bound to OnConnectLoginChanged] │ ├─► EOSCreateLobby (Permission=PublicAdvertised, MaxMembers=4, EnableVoice=true) │ └─► EOSJoinRTCRoom via GetLobbyRTCRoomName(LobbyId) │ ├─► EOSQueryFriends, EOSQueryPresence — populate HUD │ └─► EOSBeginClientSession(ClientServer) (if AntiCheat enabled) └─► forward bytes via PollOutboundServerMessages each Tick OnGameEnd / Disconnect ─► EOSLeaveRTCRoom -> EOSLeaveLobby -> EOSEndClientSession SaveGame::Save ─► UGameplayStatics::SaveGameToSlot

Project setup checklist

  1. Enable the plugin (already done in PluginCore.uproject).
  2. Fill credentials in Project Settings → Plugins → EOSpro → Credentials.
  3. Pick default login platform in Login Defaults → Default Login Platform. PC dev: Epic Games (Account Portal).
  4. (Optional) Anti-Cheat: drop base_public.cer in Build/NoRedist/, flip Enable Anti-Cheat.
  5. (Optional) Cross-platform purchases: enable OnlineSubsystemSteam / GooglePlay / IOS per platform.

The five Blueprint classes you create

Set Project Settings → Maps & Modes → Game Instance Class to BP_EOSGameInstance and Default GameMode to BP_EOSGameMode.

BP_EOSGameInstance extends UGameInstance

Owns EOS init early, wires delegates that survive level transitions, hosts player-session-wide state.

Why a GameInstance and not the GameMode? Auth tokens / Connect PUID / RTC room membership outlive map transitions. UEOSproSubsystem is a UGameInstanceSubsystem — it auto-creates inside any GameInstance.

BP graph

Event Init
  └─► UEOSproSubsystem auto-inits (settings.bAutoInitialize=true)
  └─► Get EOSpro Subsystem
        ├─► Bind Event to OnInitialized       → Custom Event "OnEOSReady"
        ├─► Bind Event to OnAuthLoginChanged  → Custom Event "OnAuthChanged"
        └─► Bind Event to OnConnectLoginChanged → Custom Event "OnConnectChanged"

Event OnConnectChanged (ProductUserId)
  ├─► If settings.bAutoBeginPlayerSession:
  │       EOSMetricsLibrary::BeginPlayerSession(DisplayName, "", "")
  ├─► If settings.bAntiCheatEnabled:
  │       EOSAntiCheatLibrary::BeginClientSession(Mode=ClientServer)
  └─► Broadcast event "OnPlayerReady" — PlayerController/HUD listen for it
BP_EOSGameMode extends AGameMode

Host-side session/lobby create + register joining players.

Event BeginPlay
  └─► If Is Server:
        ├─► EOSCreateLobby (uses settings.DefaultLobby* values)
        │     ─OnSuccess─► Store LobbyId on GameMode
        │             └─► (optional) EOSCreateSession for matchmaking layer
        └─► (clients drive their own EOSJoinLobby from PlayerController)

Event OnPostLogin (override)
  └─► If Is Server:
        ├─► EOSSessionsLibrary::RegisterPlayer(SessionName, NewPlayer.PUID)
        └─► Broadcast lobby state change

Event EndPlay
  ├─► EOSLobbyLibrary::DestroyLobby(LobbyId)
  └─► EOSSessionsLibrary::DestroySession(SessionName)
BP_EOSPlayerController extends APlayerController

Auth, voice push-to-talk, per-player UI.

Event BeginPlay
  ├─► If Is Local Controller:
  │     ├─► EOSLoginWithPlatform
  │     │     Platform: settings.DefaultLoginPlatform
  │     │     Token/Id/DisplayName: per-platform values
  │     │     ─OnSuccess─►
  │     │           ├─► Create Widget BP_EOSHUD → Add To Viewport
  │     │           └─► EOSJoinRTCRoom (room name from GetLobbyRTCRoomName)
  │     │
  │     └─► Setup Input bindings:
  │           ├─► "PushToTalk" (V) — Pressed: TransmitToAllRooms(true), Released: false
  │           ├─► "ToggleSocialOverlay" (Shift+Tab) → EOSUILibrary::ShowSocialOverlay
  │           └─► "OpenStore" (B) → push WBP_Store if EOSPlatformPurchaseAvailable

Event EndPlay
  ├─► EOSUpdateRTCSending(CurrentRoom, false)
  ├─► EOSLeaveRTCRoom(CurrentRoom)
  └─► EOSAuthLibrary::Logout

WBP_VoiceSettings Runtime voice settings menu (separate from the editor debugger). Mic/speaker combos call EOSRTCLibrary::SetAudioInput/OutputSettings; AEC/NS toggles call EOSRTCLibrary::SetRTCSetting("DisableNoiseSuppression", "true"/"false"). Save chosen device IDs to your runtime BP_EOSSaveGame (NOT to UEOSproSettings — that's an editor-config asset).

GameInstance hooks

Event OnPlayerReady (from BP_EOSGameInstance)
  └─► LoadGameFromSlot("DefaultSave", 0) → cast to BP_EOSSaveGame
        ├─► If exists:
        │     ├─► Apply Mic/Speaker DeviceId via EOSRTCLibrary::SetAudioInput/OutputSettings
        │     ├─► Apply bMicMuteOnJoin into a GameInstance-level bool
        │     └─► Restore inventory into PlayerState
        └─► If not exists:
              └─► Spawn fresh BP_EOSSaveGame with defaults

Event OnApplicationWillDeactivate (FCoreDelegates)
  └─► SaveGameToSlot(CurrentSave, "DefaultSave", 0)

Cross-platform purchase flow (OSS bridge)

The bridge auto-routes through whichever store SDK is loaded. Write the BP graph ONCE.

WBP_Store::OnConstruct
  ├─► EOSGetActivePlatformOSS         → show platform name in a debug text block
  ├─► EOSPlatformPurchaseAvailable    → if false, disable buy buttons
  ├─► EOSPlatformQueryReceipts(true)  → fill OwnedItems list
  └─► EOSQueryOffers (Epic) OR EOSQueryOffersById([IDs]) (Google Play)
        └─► populate offer cards

WBP_OfferCard::OnBuyClicked
  └─► EOSPlatformPurchase(OfferId, "", 1)
        ├─OnSuccess(Receipt)─►
        │     ├─► Add to OwnedItems
        │     ├─► Grant inventory item to player
        │     ├─► EOSPlatformFinalizePurchase(Receipt.TransactionId)
        │     │       ↳ REQUIRED on Google Play within 3 days
        │     └─► Save the save game
        └─OnFailure──► toast error
PlatformGotcha
Google PlaySkipping EOSPlatformFinalizePurchase within 3 days = auto-refund. Wire it on success unconditionally.
App StoreReceipt validation may need a server round-trip. Always call EOSPlatformQueryReceipts(true) on app launch.
SteamEOSPlatformFinalizePurchase is a no-op (Steamworks acknowledges automatically). Calling it is still safe.
Epic Games StoreEquivalent to EOSRedeemEntitlements from the EOS Ecom path. Use the OSS bridge for consistency.

Anti-Cheat wiring

EOSpro exposes a scaffold — you wire the bytes through your normal network channel.

GameMode (dedicated server) BeginPlay:
  └─► If is dedicated server and AC enabled:
        ├─► EOS_AntiCheatServer init (C++; not yet a BP node — extend subsystem)
        └─► Per-tick: drain server-side AC messages, RPC them to clients

PlayerController (client) BeginPlay (after Connect login):
  └─► EOSAntiCheatLibrary::BeginClientSession(Mode=ClientServer)
  └─► Bind Tick:
        ├─► For each msg in PollOutboundServerMessages:
        │       └─► Server RPC: SendAntiCheatBytesToServer(msg.Data)
        ├─► If P2P mode: also drain PollOutboundPeerMessages, forward to each peer

ServerRPC "SendAntiCheatBytesToClient" (server → client):
  └─► On client: EOSAntiCheatLibrary::ReceiveMessageFromServer(bytes)

PlayerController EndPlay:
  └─► EOSAntiCheatLibrary::EndClientSession
Required Drop base_public.cer from EOS Portal → Anti-Cheat → Download integrity keys into Build/NoRedist/base_public.cer. Flip bAntiCheatEnabled in settings. Click Validate Anti-Cheat Cert in Project Settings to verify it's at the right path.

Common pitfalls

SymptomCauseFix
Login node returns silent failure in PIE Missing credentials or wrong default platform Output Log for LogEOSpro. PIE → use Developer + DevAuthTool token.
Voice join succeeds but no audio bDefaultMicMuted true OR no TransmitToAllRooms(true) call Check Voice Chat settings + push-to-talk binding.
GetLobbyRTCRoomName returns empty Lobby created with bEnableRTCRoom=false Set true on EOSCreateLobby or in Settings → Lobby Defaults.
EOSPlatformPurchase "No active OnlineSubsystem" Matching engine plugin not enabled Enable OnlineSubsystemSteam/GooglePlay/IOS + set DefaultPlatformService.
BeginClientSession returns false Cert missing or Connect login not complete Validate Anti-Cheat Cert button. Gate on OnConnectChanged.
LoadGameFromSlot returns null First-time launch — no save yet Always null-check and spawn a fresh BP_EOSSaveGame.

End-to-end PIE test checklist

  1. Install Epic Games Launcher; sign in once.
  2. Unzip Source/ThirdParty/EOSSDK/DevAuth/EOS_DevAuthTool-win32-x64-1.2.1.zip, run it, log in, name your dev credential DevA on port 6547.
  3. Project Settings → EOSpro → Login Defaults → Default Login Platform = DevAuth Tool.
  4. PIE with 2 players (Number of Players = 2, Net Mode = Play As Client).
  5. Player 1 logs in with Token=DevA, Id=127.0.0.1:6547. Player 2 with Token=DevB (create in DevAuthTool first).
  6. Player 1 (host) creates the lobby; both see Player Count = 2 in HUD.
  7. Both press V (PTT) — speaking indicator lights up.
  8. Press Shift+F1 — EOS Social Overlay appears.
  9. Window → EOS Voice Chat Debugger — verify both players show in participants with live mic state.

If any step fails, cross-reference the pitfalls table above.
